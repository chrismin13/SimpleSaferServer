About Backup Drive Setup
Use this when:
  • you replaced the backup drive with a different physical disk
  • the original UUID or USB ID was detected incorrectly during setup or migration
  • you need to point the backup system at the correct backup partition again
Do not use this when:
  • you are only changing cloud backup settings or schedules
  • you are trying to manage unrelated disks or other /etc/fstab entries
  • the system is already configured correctly and the drive has not changed
What this updates:
  • the backup drive mount point in /etc/SimpleSaferServer/config.conf
  • the stored backup drive UUID and USB ID
  • the SimpleSaferServer-managed line in /etc/fstab for the backup drive
  • the Samba backup share path in /etc/samba/smb.conf if the mount point changed
What it will not do:
  • it will not modify unrelated /etc/fstab entries
  • it will stop and ask for manual cleanup if it finds conflicting or ambiguous mount entries
  • it is not a general-purpose disk management tool
What “SimpleSaferServer-managed” means:

The app only updates the backup drive line in /etc/fstab that ends with its marker comment. That marker is just the trailing comment # SimpleSaferServer managed backup drive.

Example:
UUID=2CD49023D48FED80 /media/backup ntfs-3g defaults,nofail 0 0 # SimpleSaferServer managed backup drive
